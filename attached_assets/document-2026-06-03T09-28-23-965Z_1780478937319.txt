Mandatory timeframe model
The config must explicitly use:

entry_timeframe = "1m"
screening_timeframe = "15m"
confirmation_timeframe = "5m"

Meaning:

1m is for entry and execution signals.
15m is for screening and regime bias.
5m is for confirmation.

Never infer timeframe roles from array order.
If config parsing exists, make sure it supports these three explicit fields.
If config parsing is not ready yet, make sure the core types are designed to support these three fields later.

Every Signal must have signal_id.
This is required because all later objects must trace back to the original signal:
signal_id -> order_id -> fill_id -> position_id -> exit_order_id -> trade_id
